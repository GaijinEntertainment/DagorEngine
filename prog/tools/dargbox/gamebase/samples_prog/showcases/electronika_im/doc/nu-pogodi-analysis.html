Technical and Historical Analysis of the Soviet “Nu, pogodi!” Handheld Game (Elektronika IM-02)

Game Algorithm and Egg-Generation Behavior

Despite its simple concept, “Nu, pogodi!” (like the original Egg game) implements a carefully crafted algorithm for egg generation, timing, scoring, and difficulty progression. The gameplay is as follows: Four hens (two on an upper row, two on a lower row) lay eggs that roll down four chutes (paths) toward the screen’s center. The player controls the wolf, who can move to four positions to catch eggs in his basket before they hit the ground. Key aspects of the game’s algorithm and behavior include:

  • Sequential Game Loop: The game runs in a continuous loop driven by the microcontroller’s clock. Each loop cycle corresponds to a small time step; after a certain number of cycles (a “wait loop” threshold), an egg movement/update happens. Initially, the delay between updates is relatively long, making eggs fall slowly. As the score increases, this delay shortens (eggs fall faster), up to a minimum cap to keep the game human-playable[25][26]. In the code, a counter wait_loop_counter increments each CPU cycle and the game logic update triggers when it exceeds wait_loop_max (which starts at a high value and decreases with difficulty)[27][28]. This effectively throttles the game speed.

  • Egg Movement: On each update step, every egg on screen advances one position down its respective chute. Each chute has 5 discrete positions (segments for the egg graphic). The data for egg positions can be visualized as four arrays (one per chute) of five slots[29][30]. An egg in the 5th (last) position is at the brink of falling to the floor. The game logic checks for this condition: if an egg reaches the end of a chute, it is flagged as an “egg in danger” for that chute[31]. This means that on the next update, if the wolf is not in position to catch it, it will drop and count as a miss.

  • Egg Generation Algorithm: The crux of the gameplay is the generation of new eggs from the hens. The algorithm uses a pseudo-random selection process combined with deterministic rules to decide when and where a new egg will appear. In each cycle where an update occurs, the program picks a current hen (one of the four) – in Game B this is chosen at random, whereas in Game A there are some constraints (described below) on which hen can lay eggs at a given time[32]. A random number generator (seeded by a free-running hardware timer) produces a value 0–15 each cycle[33], and this is used to derive a random hen index and a random boolean condition. Specifically, the logic does something equivalent to:

  • Select a Hen: Compute a random hen index 0–3 (e.g. via random_number mod 4) for the next potential egg[34]. This defines the “current hen” whose chute will be considered for egg dropping in this loop. (On some cycles, especially in Game A, this hen might be overridden or ignored – see below.)

  • Determine Egg Drop: If the chosen hen’s chute is clear at the start (no egg currently in the first position) and the total number of eggs on screen is below the allowed maximum, the game will attempt to spawn a new egg. A random boolean is used to decide whether an egg is actually laid this cycle[35]. In pseudocode from the extracted ROM, the condition is roughly: if (RandomBool == TRUE) or (no eggs have been spawned yet) then lay a new egg in current hen’s first position[36]. The second condition ensures the game always starts by spawning an egg if none exist (so the game isn’t stuck in a no-egg state).

  • Spawn and Tracking: When an egg is “laid,” the first segment of that hen’s chute is activated (an egg appears at the top of the chute) and a global counter of active eggs (spawned_egg_counter) is incremented[36]. The newly spawned egg will begin moving down on subsequent updates.

The pseudo-random generator in these Elektronika games is very primitive – it essentially samples the low bits of an internal timer to get a semi-random sequence[33]. This means the pattern of eggs, while unpredictable to the player, is determined by the initial state and timing of the device. Reverse-engineered code confirms that the random function returns both a 4-bit random number and a boolean flag used for decisions[33].

  • Game A vs Game B (Difficulty Modes): As with the Nintendo original, Nu, pogodi! offers two game modes. Game A is the easier mode, and Game B is a faster, more difficult mode. The primary difference lies in the egg generation pattern:

  • In Game A, at any given time only three of the four chutes are active[25][32]. One hen is effectively “idle,” not laying eggs, which reduces the cognitive load on the player. Interestingly, the inactive chute in Game A is not fixed; it depends on the number of misses (penalty points) the player currently has. When the game starts (0 misses), the bottom-left chute will produce no eggs[37]. If the player incurs a miss (or half-miss), the game switches which chute is inactive: for 0.5 or 1.0 miss, the bottom-right chute goes idle; at 1.5–2 misses, the top-left goes idle; and at 2.5 misses (i.e. a third miss in blink state), the top-right becomes inactive[37]. In essence, whenever misses are added or cleared, the algorithm rotates which hen is being ignored, always keeping only 3 active. This clever mechanism gives a slight break to the player after each mistake (by removing the chute where the mistake occurred or balancing the load), while also cycling through all chutes over the course of the game. The result is that Game A is forgiving and somewhat patterned, never throwing eggs from all four directions at once[25].

  • In Game B, all four chutes are active all the time, and eggs can come from any of the four hens at random[25]. There is no “ignore hen” rule in this mode[32]. This effectively increases the maximum on-screen chaos and makes the game more challenging. Additionally, Game B typically starts at a slightly faster base speed. In the code, for example, the initial wait-loop delay is set shorter for Game B (e.g. wait_loop_max = 25 in Game B vs 31 in Game A)[38], causing eggs to move a bit quicker even at the outset.

  • Difficulty Progression (Egg Speed & Quantity): Nu, pogodi! gradually increases the difficulty as the player scores points, both by speeding up the egg drops and by allowing more eggs on screen simultaneously:

  • Speed Increase: The interval between egg movements shortens as score rises. Internally, every time the player’s tens digit in the score changes (i.e. every 10 points), the game makes the eggs fall slightly faster by reducing the delay counter by 1 (down to a minimum threshold)[26]. This means the game smoothly accelerates throughout each 0–99 point span. However, at each hundred-point milestone (100, 200, etc.), the game gives a brief reprieve: the speed is slightly reduced again, making the eggs fall slower immediately after crossing the hundred[25][39]. In fact, the code shows that on crossing a hundred (except 900→0 rollover), the delay counter is increased by a fixed amount (e.g. 7 units) to significantly slow the pace[40][28]. This creates a momentary breather for the player before the speed ramps up again. For example, if you reach 100 points, the speed almost resets to what it was at the game start, then begins increasing anew[25]. These “slowdown” events at 100, 200, 300, etc., were also present in Nintendo’s original game as a reward mechanism. By 900+ points, the game is extremely fast; after 999 it rolls over to 000 but retains the high speed (slowing only a little)[10], making the second loop even more challenging.

  • Concurrent Eggs: The game limits how many eggs can be in play at the same time, and this limit rises with the score. At the very start, only 1 or 2 eggs can be rolling at once. But as the player proves skill by scoring points, the internal max_eggs value increases stepwise. Reverse-engineered logic shows that max_eggs is computed from the score’s higher digits[41][42]. In general, after a few points it becomes 3, then 4, 5, and so on. For instance, once the player scores 5 points, the game may allow up to 4 eggs concurrently on screen; after 9 points, up to 5 eggs; after 11 points, 7 eggs; and ultimately it can allow well over 9 eggs at once at the highest difficulties[43][44]. (The exact thresholds in the code use a formula based on the sum of the tens and hundreds digits of the score[45].) In practical terms, the higher the score, the more overlapping eggs the player must juggle, which dramatically increases difficulty. In the absolute hardest phase, the limit max_eggs reaches 12 concurrent eggs[41], though in normal play the effective number rarely gets that high before a miss occurs.

Together, the increasing speed and number of eggs ensure that the game becomes very challenging at high scores, preventing players from playing indefinitely. As noted above, the game does not truly end at 999 points – it wraps around, but by then the speed is so high that surviving beyond 1000 is mostly theoretical. (The high-score memory will still display “999” as the record[10].)

  • Scoring and Misses: Each caught egg awards 1 point to the score[46]. If the wolf fails to catch an egg, it “falls” and breaks on the ground – this counts as a miss (a penalty). Misses are visually indicated by a small broken egg or chick icon on the screen. Uniquely, Nu, pogodi! (and the Egg game) implement a concept of half-misses under certain conditions:

  • A standard miss (egg falls with no special condition) gives a full penalty. The LCD will display a static broken egg/chick symbol (often at either the left or right bottom corner, depending on which side was missed) to represent that miss. Miss counter increases by 1.

  • If a miss happens while the hare is present on screen, the player is penalized with only half a miss[5]. In this case, instead of a full broken egg, the LCD shows a little chick hatching from the broken egg and that icon blinks to indicate the half-miss state[5]. What is happening in-game is that the hare (Zayats) occasionally peeks out of a window in the top-left corner of the screen to “distract” the wolf. If the player misses an egg during that moment, the game treats it more leniently: internally a half-miss is recorded (the code sets a flag half_miss = true)[47][5]. This means the miss hasn’t fully counted against the 3-miss limit yet.

The half-miss will eventually turn into a full miss if the player makes another mistake or after some time passes. In the program’s logic, a half miss is basically a miss that is awaiting confirmation. If the player does not drop another egg immediately, the half miss will “finalize” as a full miss after a short interval (the chick icon stops blinking and stays on, half_miss flag cleared)[48]. However, if another egg is missed while a half-miss is still blinking, that half-miss and the new miss might combine to count as one and a half or two misses total (depending on sequence)[49][32]. This system is a bit complex, but effectively it gives the player one extra hit buffer: the very first miss in presence of the hare only counts as 0.5, so the player could technically miss 4 times (if one is half) before game over. Thematically, the hare’s presence “softens” the penalty – a nice touch connecting to the cartoon (where the hare always outwits the wolf).

  • The game ends when the player accumulates 3 full miss points[50]. On the third full miss, a game-over sound plays and the game resets to clock/demo mode[50]. Due to half-miss mechanics, it’s possible to drop 5 or 6 eggs before losing (for example, 2 full misses + 2 half-misses count as 3 total). The Soviet manual describes this as “from three to six fallen eggs ends the game” because of the half-point system[51]. After game over, the score display resets and the player can start a new game.

  • There is a miss reset bonus: upon reaching 200 points and 500 points, any accumulated misses are cleared (all penalty icons removed)[51]. This is another element inherited from Nintendo’s design (many G&W games forgive misses at certain score milestones). It allows skilled players who last that long to continue without the burden of earlier mistakes. Notably, the half-miss state is also cleared at these checkpoints. The Soviet documentation confirms that at 200 and 500 caught eggs, the “penalty points are annulled”[51]. After 999 and rollover, misses are not automatically cleared (only another 200 points into the next loop would clear them again)[10].

  • Hare Appearance and Idle Periods: One subtle aspect of the algorithm is how and when the hare (Zayats) appears in the window – which correlates with periods of no eggs being spawned. The program keeps an idle_hen_counter to track consecutive cycles where no new egg was laid[52]. If eggs haven’t been generated for a few cycles in a row (meaning a brief lull in gameplay), the hare will pop out, presenting the distraction opportunity. In code, after 3 consecutive “idle” cycles the loop breaks and likely triggers the hare’s appearance routine (this matches anecdotal gameplay where the hare shows up during slower moments)[52][53]. As soon as an egg is laid, the idle counter resets and the hare retreats. This interplay ensures that the hare (and the chance for half-miss) tends to appear when the action is relatively calm – providing a kind of catch-up mechanic or at least a dynamic visual element. It’s an interesting design that adds personality to the game without altering the core egg-catching mechanic.

In summary, the game algorithm for “Nu, pogodi!” is a faithful replication of the Game & Watch Egg logic with a few thematic enhancements. The pseudo-code of the original ROM (reconstructed by enthusiasts) clearly shows the structured loop: choose hen -> shift eggs -> possibly spawn new egg (if random and under limit) -> check catches/misses -> update score and speed[35][54]. The Soviet clone did not fundamentally change the logic, aside from integrating the hare character for the half-miss feature (Nintendo’s Mickey version had a similar idea with Minnie Mouse, so this too was likely analogous to the original). All of these behaviors – random egg drops, increasing speed, limited concurrent eggs, the miss system – work in concert to create a game that starts easy and becomes frenetic, challenging the player’s reflexes and concentration.
